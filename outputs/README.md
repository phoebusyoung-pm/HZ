# 展务系统静态 HTML 原型

直接双击 `dashboard.html` 可进入 PC 企业端；双击 `admin-dashboard.html` 可进入 PC 后台管理端；双击 `mini-home.html` 可进入小程序模拟端。

## 文件结构

- PC 企业端：企业资料已合并为“参展资料”；绿搭与光地页面内置方案、改稿、名片及报到通知书流程
- PC 后台端：新增绿搭业务详情、光地业务详情和系统设置页面
- 小程序端：底部导航为“首页、展位业务、进度、证件”，新增 `mini-booth.html`

## 最新交互说明

- 企业端业务页面默认处于查看状态，点击底部“编辑”后进入编辑状态，并显示“保存草稿、提交审核”操作。
- 参展资料中的企业基础资料、企业及展品、宣传资料在同一页面连续展示。
- 报到通知书包含报到时间、地点、展位号、报到流程、注意事项及现场对接人。
- 后台消息通知支持短信模板配置与短信推送模拟。
- 系统设置中的绿搭模板可配置面积、上传画面数量和画面名称。
- 公共样式：`style.css`
- 公共交互：`app.js`
- 展位视觉素材：`assets/green-booth.png`

所有页面均为纯前端静态原型，不依赖框架或接口。上传、下载、审核、弹窗、标签页和状态变化均为模拟交互。

旧的 `product-info.html`、`brand-info.html` 和独立通知书入口仍保留，用于兼容已有链接。
