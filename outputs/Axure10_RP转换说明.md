# Axure 10 RP 转换说明

当前交付物为纯前端静态 HTML 原型，入口位于 `outputs` 目录。

## 关于 `.rp` 格式

Axure `.rp` 是 Axure RP 的专有工程格式，不是公开的 HTML / XML / JSON 格式。  
在未安装 Axure、也没有官方可调用转换能力的情况下，无法保证把现有 HTML 自动转换为“可在 Axure 10 内编辑”的 `.rp` 工程文件。

请不要直接把 `.html` 或 `.zip` 改名为 `.rp`，否则 Axure 可能无法打开，或打开后不是可编辑组件。

## 当前可保证的内容

当前 HTML 原型已按以下方式处理：

- 页面统一使用 UTF-8 编码。
- 页面布局为固定后台管理风格，PC 端左侧菜单 + 顶部栏 + 内容区。
- 小程序端为移动端卡片布局。
- 按钮、表格、弹窗、标签页、状态标签均为静态模拟交互。
- 直接用浏览器打开时，可保持当前界面位置与按钮布局。

## 如果必须交付 Axure 10 可编辑 `.rp`

需要采用以下方式之一：

1. 在安装 Axure 10 的电脑上，由设计人员按当前 HTML 原型逐页重建为 Axure 元件。
2. 提供现有 Axure `.rp` 模板文件，由设计人员在 Axure 内粘贴/重建页面结构。
3. 若只要求演示不要求编辑，可继续使用当前 HTML 原型或将页面截图作为 Axure 图片页面，但图片方式无法编辑按钮和文本。

## 防乱码建议

如果后续在 Axure 中重建：

- 所有页面文字统一使用中文字体：微软雅黑 / Microsoft YaHei。
- 导入或复制文字时确保源文件为 UTF-8。
- 避免从乱码终端复制文本。

## 防位移建议

如果后续在 Axure 中重建：

- PC 页面建议画布宽度：1440px。
- 左侧菜单宽度：238px。
- 顶部栏高度：68px。
- 内容区左边距：26px。
- 卡片圆角：12px。
- 表格行高：56px 左右。
- 小程序页面建议设备宽度：390px。
